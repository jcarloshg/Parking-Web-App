# Specification Software Document (SSD)

## 1. Introducción

### 1.1 Propósito

Este documento especifica los requisitos técnicos y funcionales del sistema de gestión de estacionamiento público.

### 1.2 Alcance

Sistema web full stack con autenticación JWT para gestionar: entradas/salidas de vehículos, tarifas automáticas, cajones disponibles, reportes de ingresos y administración de usuarios.

---

## 2. Arquitectura

### 2.1 Stack Tecnológico

- **Backend**: Laravel 10+
- **Frontend**: Vue.js 3 (Composition API)
- **Base de datos**: MySQL
- **Autenticación**: JWT (tymon/jwt-auth)
- **Control de versiones**: Git

### 2.2 Estructura de Directorios

```
parking-web-app/
├── backend/          # Laravel API
├── frontend/         # Vue.js SPA
└── docs/             # Documentación
```

## 3. Base de Datos

### 3.1 Esquema de Tablas

#### users

| Campo          | Tipo                                  | Descripción     |
| -------------- | ------------------------------------- | --------------- |
| id             | BIGINT UNSIGNED                       | PK              |
| name           | VARCHAR(255)                          | Nombre completo |
| email          | VARCHAR(255)                          | Email único     |
| password       | VARCHAR(255)                          | Hash bcrypt     |
| role           | ENUM('admin', 'cajero', 'supervisor') | Rol del usuario |
| remember_token | VARCHAR(100)                          | Token remember  |
| created_at     | TIMESTAMP                             |                 |
| updated_at     | TIMESTAMP                             |                 |

#### parking_spaces (Cajones)

| Campo      | Tipo                                            | Descripción             |
| ---------- | ----------------------------------------------- | ----------------------- |
| id         | BIGINT UNSIGNED                                 | PK                      |
| number     | VARCHAR(10)                                     | Número de cajón (único) |
| type       | ENUM('general', 'discapacitado', 'eléctrico')   | Tipo de cajón           |
| status     | ENUM('disponible', 'ocupado', 'fuera_servicio') | Estado actual           |
| created_at | TIMESTAMP                                       |                         |
| updated_at | TIMESTAMP                                       |                         |

#### tickets (Registro de vehículos)

| Campo            | Tipo                              | Descripción        |
| ---------------- | --------------------------------- | ------------------ |
| id               | BIGINT UNSIGNED                   | PK                 |
| plate_number     | VARCHAR(20)                       | Placa del vehículo |
| vehicle_type     | ENUM('auto', 'moto', 'camioneta') | Tipo de vehículo   |
| entry_time       | DATETIME                          | Hora de entrada    |
| exit_time        | DATETIME NULL                     | Hora de salida     |
| parking_space_id | BIGINT UNSIGNED FK                | Cajón asignado     |
| status           | ENUM('activo', 'finalizado')      | Estado del ticket  |
| created_at       | TIMESTAMP                         |                    |
| updated_at       | TIMESTAMP                         |                    |

#### payments (Pagos)

| Campo          | Tipo                        | Descripción        |
| -------------- | --------------------------- | ------------------ |
| id             | BIGINT UNSIGNED             | PK                 |
| ticket_id      | BIGINT UNSIGNED FK          | Ticket relacionado |
| total          | DECIMAL(10,2)               | Total a pagar      |
| payment_method | ENUM('efectivo', 'tarjeta') | Método de pago     |
| paid_at        | DATETIME                    | Fecha de pago      |
| created_at     | TIMESTAMP                   |                    |
| updated_at     | TIMESTAMP                   |                    |

### 3.2 Índices

- users: email (único)
- parking_spaces: number (único)
- tickets: plate_number, status, parking_space_id
- payments: ticket_id

## 4. Autenticación JWT

### 4.1 Paquete

- **tymon/jwt-auth** para Laravel

### 4.2 Endpoints de Autenticación

| Método | Endpoint           | Descripción               |
| ------ | ------------------ | ------------------------- |
| POST   | /api/auth/login    | Iniciar sesión            |
| POST   | /api/auth/register | Registrar usuario (admin) |
| POST   | /api/auth/logout   | Cerrar sesión             |
| POST   | /api/auth/refresh  | Refresh token             |
| GET    | /api/auth/me       | Datos usuario actual      |

### 4.3 Configuración JWT

- **TTL**: 60 minutos
- **Refresh TTL**: 20160 minutos (2 semanas)
- **Algoritmo**: HS256

### 4.4 Headers de Solicitud

```
Authorization: Bearer <token_jwt>
```

## 5. Reglas de Negocio

### 5.1 Tarifas de Estacionamiento

| Tipo Vehículo | Tarifa Hora | Tarifa Diaria |
| ------------- | ----------- | ------------- |
| auto          | $20.00      | $150.00       |
| moto          | $10.00      | $80.00        |
| camioneta     | $30.00      | $200.00       |

### 5.2 Condiciones de Cobro

- **Tolerancia**: 10 minutos gratis (sin costo)
- **Cobro**: Por hora completa
- **Más de 24h**: Aplicar tarifa diaria

### 5.3 Roles y Permisos

| Acción             | Admin | Cajero | Supervisor |
| ------------------ | ----- | ------ | ---------- |
| Registrar entrada  | ✓     | ✓      | ✓          |
| Registrar salida   | ✓     | ✓      | ✓          |
| Ver reportes       | ✓     | ✗      | ✓          |
| Gestionar usuarios | ✓     | ✗      | ✗          |
| Gestionar cajones  | ✓     | ✗      | ✗          |

## 6. API Endpoints

### 6.1 Autenticación

```
POST   /api/auth/login
POST   /api/auth/register
POST   /api/auth/logout
POST   /api/auth/refresh
GET    /api/auth/me
```

### 6.2 Tickets

```
GET    /api/tickets              # Listar tickets (paginación)
GET    /api/tickets/{id}         # Ver ticket
POST   /api/tickets              # Crear ticket (entrada)
PUT    /api/tickets/{id}/exit    # Registrar salida
GET    /api/tickets/active       # Tickets activos
GET    /api/tickets/search       # Buscar por placa
```

### 6.3 Payments

```
GET    /api/payments             # Listar pagos
GET    /api/payments/{id}        # Ver pago
POST   /api/payments             # Crear pago
GET    /api/payments/today       # Pagos de hoy
```

### 6.4 Parking Spaces

```
GET    /api/parking-spaces              # Listar cajones
GET    /api/parking-spaces/{id}         # Ver cajón
POST   /api/parking-spaces              # Crear cajón
PUT    /api/parking-spaces/{id}         # Actualizar cajón
DELETE /api/parking-spaces/{id}         # Eliminar cajón
GET    /api/parking-spaces/available    # Cajones disponibles
```

### 6.5 Usuarios (Admin)

```
GET    /api/users              # Listar usuarios
GET    /api/users/{id}         # Ver usuario
POST   /api/users              # Crear usuario
PUT    /api/users/{id}         # Actualizar usuario
DELETE /api/users/{id}         # Eliminar usuario
```

### 6.6 Reportes

```
GET    /api/reports/daily      # Reporte diario
GET    /api/reports/monthly   # Reporte mensual
GET    /api/reports/summary    # Resumen dashboard
```

## 9. Validación

### 9.1 Reglas por Endpoint

#### Login

- email: required|email
- password: required|string

#### Registro Ticket

- plate_number: required|string|regex:/^[A-Z0-9-]+$/i
- vehicle_type: required|in:auto,moto,camioneta
- parking_space_id: required|exists:parking_spaces,id

#### Registro Pago

- ticket_id: required|exists:tickets,id
- total: required|numeric|min:0
- payment_method: required|in:efectivo,tarjeta
